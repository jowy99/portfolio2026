## Résumé

Monopoly est une version basique du jeu de plateau classique, implémentée pour fonctionner en terminal. Le projet a été réalisé comme travail académique et a respecté les exigences fonctionnelles du cours.

Même si le travail était prévu pour trois personnes, j'ai assuré toute l'implémentation de manière individuelle.

## Objectifs

1. Construire un jeu console fonctionnel en Java.
2. Respecter les critères académiques du devoir.
3. Appliquer la programmation orientée objet dans un cas concret.
4. Livrer un résultat stable dans le délai imparti.

## Défis

Le principal défi a été de travailler avec peu de guidance pédagogique sur la structure du projet et les bases Java. Ce manque de contexte m'a obligé à résoudre les questions techniques par recherche et expérimentation.

Il y avait aussi une contrainte de temps en raison d'une charge initialement répartie entre plusieurs personnes.

## Décisions techniques

J'ai opté pour une architecture simple et modulaire pour séparer logique de jeu, tours et état de partie. L'objectif était de garder un code lisible et facile à faire évoluer.

L'implémentation est restée entièrement en terminal, sans interface graphique, pour prioriser la stabilité et les mécaniques essentielles.

## Points forts

- Jeu exécutable en terminal avec gestion des tours.
- Implémentation d'une rue de Monopoly et des mécaniques de base.
- Code construit intégralement par une seule personne.
- Référentiel public pour revue technique.

## Résultats / Impact

Le projet a été livré et fonctionne dans le cadre académique. Il a permis de démontrer capacité de livraison et résolution autonome malgré un accompagnement limité.

Sur le plan pratique, j'ai consolidé mes bases en Java et ma confiance pour aborder des projets techniques avec peu de supervision.

## Ce que j'ai appris / Ce que j'améliorerais

J'ai appris qu'une organisation précoce du code et une documentation minimale sont très utiles quand le contexte est flou. J'ai aussi confirmé qu'avancer par petites étapes accélère la progression dans les projets d'apprentissage.

Dans une prochaine itération, j'étendrais les règles du jeu, j'ajouterais davantage de cas de test et j'améliorerais l'expérience d'entrée en console.
