## Summary

Memory Game is a simple application designed for light entertainment while training memory. It is based on a fast and intuitive mechanic, focused on a smooth and pleasant experience.

I developed the project from scratch, including both design and frontend implementation.

## Objectives

1. Build a simple and accessible game for any user.
2. Implement clear and stable memory logic.
3. Keep a minimalist, clean, and functional interface.
4. Deliver a fluid user experience.

## Challenges

There were no major technical blockers because of the limited scope. The main focus was keeping logic organized and avoiding unnecessary complexity.

I also prioritized game-flow clarity so users understand the dynamics from the first try.

## Technical Decisions

The implementation used JavaScript for interactive logic, HTML for semantic structure, and TailwindCSS for a consistent visual layer.

I prioritized a simple architecture to keep maintenance and future evolution straightforward.

## Highlights

- Simple, functional interface focused on usability.
- Intuitive mechanic for a broad range of users.
- Lightweight game ideal for short play sessions.
- Full implementation completed by one developer.

## Results / Impact

The project is complete and meets its goal of lightweight entertainment with a stable experience. It provided practical training in JavaScript interaction patterns.

It also reinforced minimalist design criteria focused on essentials: clarity, responsiveness, and ease of use.

## What I Learned / What I'd Improve

This project strengthened my JavaScript skills for game logic. It also confirmed the value of simple designs when speed and user experience are priorities.

In a next iteration, I would add difficulty levels, time scoring, and basic game statistics.
