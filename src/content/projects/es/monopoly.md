## Resumen

Monopoly es una version basica del clasico juego de mesa, implementada para ejecutarse en terminal. El proyecto se realizo como tarea academica y cumplio los requisitos funcionales definidos para la asignatura.

Aunque el trabajo estaba planteado para un equipo de tres personas, asumi toda la implementacion de forma individual y desarrolle el proyecto desde cero.

## Objetivos

1. Construir un juego funcional en Java orientado a consola.
2. Cumplir con los criterios academicos de la practica.
3. Aplicar conceptos de programacion orientada a objetos en un caso real.
4. Entregar un resultado estable dentro del plazo disponible.

## Retos

El reto principal fue trabajar con poca guia docente sobre estructura de proyecto y conceptos base de Java. Esa falta de contexto obligo a resolver dudas tecnicas mediante investigacion y prueba iterativa.

Tambien hubo presion de tiempo al asumir una carga que originalmente se repartia entre varias personas.

## Decisiones tecnicas

Opte por una arquitectura simple y modular para separar logica de juego, turnos y estado de la partida. El objetivo fue mantener el codigo legible y facilitar cambios sin romper el flujo principal.

La implementacion se centro al 100% en terminal, sin interfaz grafica, para priorizar funcionalidad, reglas basicas y estabilidad.

## Highlights

- Juego ejecutable en terminal con flujo de turnos.
- Implementacion de una calle de Monopoly y mecanicas basicas de interaccion.
- Codigo construido de forma integra por una sola persona.
- Repositorio publico para revision tecnica del proyecto.

## Resultados / Impacto

El proyecto quedo terminado y funcional dentro del marco academico. Permitio demostrar capacidad de entrega y resolucion autonoma incluso con orientacion limitada.

Como resultado practico, consolide fundamentos de Java y gane confianza para abordar proyectos tecnicos sin depender de guia constante.

## Lo que aprendi / Que haria mejor

Aprendi que la organizacion temprana del codigo y la documentacion minima ayudan mucho cuando el contexto es difuso. Tambien confirme que dividir problemas en pasos pequenos acelera el avance en proyectos de aprendizaje.

En una siguiente iteracion ampliaria reglas del juego, anadiria mas casos de prueba y mejoraria la experiencia de entrada por consola.
