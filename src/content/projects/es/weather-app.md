## Resumen

Weather App es una aplicacion del tiempo con multiples funcionalidades para consultar el clima de cualquier ciudad en tiempo real. La interfaz sigue una linea moderna, accesible y minimalista para facilitar el uso diario.

El proyecto fue desarrollado completamente por mi con React, Vite y TailwindCSS, integrando datos meteorologicos desde weatherapi.com.

## Objetivos

1. Construir una aplicacion meteo clara, rapida y responsive.
2. Practicar consumo de APIs externas y gestion de estado en React.
3. Mejorar experiencia de usuario con funcionalidades utiles en uso real.
4. Consolidar buenas practicas de UI y rendimiento.

## Retos

Uno de los retos principales fue optimizar la gestion de ubicacion del usuario y el cache de datos meteorologicos para reducir peticiones a la API sin perder frescura de la informacion.

Tambien trabaje en accesibilidad, adaptacion responsive y pequenos detalles de interaccion para que la experiencia fuera consistente en movil y escritorio.

## Decisiones tecnicas

Use React para la arquitectura de componentes, Vite para un entorno de desarrollo rapido y TailwindCSS para iterar diseno de forma eficiente.

Implemente almacenamiento de ubicaciones favoritas (hasta 5) y atajos de teclado para acelerar la busqueda de ciudades.

## Highlights

- Atajos de teclado para busqueda rapida.
- Guardado de hasta 5 ubicaciones favoritas.
- Cambio dinamico entre modo claro y oscuro.
- Pronostico con datos detallados de humedad, presion y condiciones.
- Diseno optimizado para movil y escritorio.

## Resultados / Impacto

Aunque existen muchas apps meteo, este proyecto cumplio su proposito como laboratorio real de aprendizaje. Me permitio profundizar en React, consumo de APIs y decisiones de UX orientadas a uso cotidiano.

El resultado es una aplicacion funcional y mantenible que me sirvio para subir nivel tecnico en frontend moderno.

## Lo que aprendi / Que haria mejor

Aprendi a gestionar mejor estado y cache en interfaces con datos en tiempo real, y a tomar decisiones de producto pequenas que mejoran mucho la usabilidad.

En una iteracion siguiente anadiria pruebas automatizadas de flujos criticos y una estrategia mas completa de manejo de errores de red.
