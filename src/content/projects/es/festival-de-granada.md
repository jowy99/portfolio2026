## Resumen

Festival de Granada es una aplicacion creada para gestionar base de datos de forma mas eficiente dentro de la organizacion donde realice mis practicas. Nacio para reemplazar el uso de Excel en el manejo de informacion delicada y mejorar seguridad, consulta y automatizacion.

Ademas de proponer el cambio de enfoque, desarrolle la aplicacion completa desde cero.

## Objetivos

1. Sustituir hojas Excel por un sistema mas seguro y mantenible.
2. Facilitar la consulta y gestion de datos sensibles para empleados.
3. Reducir dependencia operativa de una unica persona.
4. Aportar una interfaz intuitiva para el uso diario del equipo.

## Retos

Uno de los retos principales fue trabajar con Laravel en una etapa temprana de aprendizaje. Hubo funcionalidades que exigieron investigacion adicional, como ordenacion alfabetica y estructuracion de consultas de forma limpia.

A traves de aprendizaje continuo y pruebas iterativas, consegui resolver esos puntos y cerrar una version estable.

## Decisiones tecnicas

La solucion se implemento con Laravel para logica de aplicacion, MySQL para persistencia y TailwindCSS para interfaz clara y rapida de mantener.

Priorice un panel interno simple, centrado en tareas reales del equipo, para que la adopcion fuera inmediata.

## Highlights

- Reemplazo efectivo de un proceso manual basado en Excel.
- Interfaz intuitiva para consulta de informacion sensible.
- Herramienta privada para uso interno de empleados.
- Mejora clara en organizacion y velocidad de trabajo.

## Resultados / Impacto

La aplicacion aporto mas productividad y autonomia al equipo de practicas. Al centralizar la informacion en una herramienta dedicada, se redujeron fricciones y se ordeno el flujo operativo diario.

El feedback fue muy positivo tanto por parte del equipo como de companeros, destacando utilidad real y calidad de la entrega.

## Lo que aprendi / Que haria mejor

Este proyecto reforzo mis fundamentos en Laravel y mi capacidad para resolver problemas con contexto cambiante. Tambien aprendi el valor de construir herramientas adaptadas a necesidades reales, no solo tecnicamente correctas.

En una siguiente iteracion anadiria trazabilidad de cambios, permisos por rol mas granulares y un set basico de pruebas automatizadas en los flujos criticos.
